# GRE 3000 词离线桌面应用设计

**日期：** 2026-07-14  
**状态：** 已在聊天中确认功能范围，等待书面规格复核  
**目标平台：** Windows 10/11 x64  
**交付形式：** 可双击运行、无控制台窗口的单文件 `.exe`

## 1. 目标

把用户提供的 `张巍GRE镇考3000词-乱序（2026年）.pdf` 转换为一个仅供个人离线学习使用的 Windows 桌面应用。应用保留词表内容与核心学习体验，但不依赖微信、账号、服务器或原 PDF 才能运行。

应用暂定名称为“GRE 3000 词离线版”。界面使用独立的绿色学习主题，不复制原小程序的商标、二维码、广告或下载入口。

## 2. 已确认需求

- 使用 Python、PySide6 和 SQLite 实现。
- 导入 PDF 中正文与补充部分的全部可识别词条。
- 支持单词搜索、连续浏览、按 PDF 顺序浏览、随机浏览、生词本和本地进度。
- 不设置每日学习量，也不强制中断学习。
- 学习页提供两种可随时切换的模式：
  - 阅读模式：完整释义始终可见。
  - 回忆模式：默认隐藏释义，单击或按空格显示。
- 两种模式共享当前单词、收藏状态和浏览位置；切换模式不会跳词。
- 通过 Windows 英文语音朗读单词。
- 记住上次使用的模式、顺序、当前位置和窗口设置。
- 应用完全离线，不包含账号、同步、遥测或网络请求。

## 3. 不在首版范围内

- 测试题、成绩记录和题目解析。
- 每日目标、打卡、连续天数或提醒。
- 个人资料、登录、云同步和多设备同步。
- 资料下载、课程、广告、公众号和二维码入口。
- PDF 未提供的记忆方法、真人音频或视频。
- 在应用内重新解析任意 PDF 或编辑词库内容。
- 自动更新器和安装程序；首版直接交付可运行的单文件 `.exe`。

## 4. 系统架构

系统分为构建阶段和运行阶段，避免把复杂的 PDF 解析逻辑带入最终应用。

### 4.1 构建阶段

`PDF -> 版面解析器 -> 规范化与校验 -> words.db + 审计报告`

版面解析器使用 PyMuPDF 的 span 级坐标信息读取 PDF。它根据稳定的五列几何位置恢复单词、音标、释义、等价词和例句，而不依赖容易错序的线性复制文本。

解析器执行以下处理：

- 忽略封面、页眉、页脚、水印、列表标题和空白行。
- 以第一列的英文词头作为每条记录的行锚点。
- 按横坐标把文本分配到五个字段，并按纵横坐标恢复阅读顺序。
- 合并英文或中文的软换行，谨慎修复被拆开的英文单词。
- 分离英文释义与中文释义、英文例句与中文译文。
- 保留原始单元格文本、PDF 页码和源顺序，便于回查。
- 对缺少词头、音标异常、跨列文本、异常高度或无法可靠分语种的记录设置质量标记。

解析结果写入只读内容数据库 `words.db`，同时生成机器可读的审计文件和便于人工检查的 HTML 报告。所有带质量标记的记录必须检查；此外随机抽查至少 100 条未标记记录。

### 4.2 运行阶段

`PySide6 界面 -> 应用服务 -> 只读 words.db / 可写 user.db / Windows 语音`

运行阶段包含以下边界清晰的组件：

- **AppShell：** 创建主窗口、页面导航、全局快捷键和主题。
- **WordRepository：** 只负责查询内置词库，不包含界面状态。
- **StudySession：** 管理顺序或随机队列、当前位置、上一词和下一词。
- **StudyViewModel：** 管理阅读/回忆模式、答案是否显示和当前词的展示状态。
- **FavoritesService：** 管理生词本。
- **ProgressStore：** 保存当前位置、浏览次数、最近浏览时间和设置。
- **SpeechService：** 封装 Qt TextToSpeech，并优先选择可用的英文 Windows 语音。
- **SearchService：** 提供大小写不敏感的前缀和包含搜索。

各组件通过明确的方法和数据对象通信。界面不直接执行 SQL，数据访问层也不依赖具体控件。

## 5. 数据设计

### 5.1 内容数据库 `words.db`

内容数据库随应用发布并保持只读。核心表 `words` 包含：

- `id`：稳定整数主键。
- `source_order`：PDF 中的源顺序。
- `source_section`：正文列表或补充列表名称。
- `source_page`：PDF 原页码。
- `headword`：单词或短语。
- `phonetic`：音标。
- `definition_en`：英文释义。
- `definition_zh`：中文释义。
- `synonyms`：保持原顺序的等价词文本。
- `example_en`：英文例句。
- `example_zh`：中文译文。
- `raw_definition`、`raw_example`：用于回查的原始单元格文本。
- `quality_flags`：解析质量标记；最终发布版允许保留已核验标记，但不得存在未复核记录。

索引覆盖 `headword` 和 `source_order`。词量只有约三千余条，不引入全文搜索扩展；普通索引和参数化查询足以提供即时搜索。

### 5.2 用户数据库 `user.db`

用户数据库通过 Qt 标准应用数据路径保存在当前 Windows 用户目录，不放在 `.exe` 所在目录。这样即使应用位于只读文件夹，仍能保存进度，替换新版 `.exe` 也不会覆盖用户数据。

核心数据包括：

- 每个词是否收藏、浏览次数、最后浏览时间。
- 当前浏览顺序、当前词 ID 和随机队列种子。
- 上次使用的阅读或回忆模式。
- 窗口大小、位置和语音速度。

数据库写入使用短事务。退出时额外刷新当前会话状态，但正确性不依赖正常退出。

## 6. 页面与交互

### 6.1 首页

- 显示应用名称、词库总数、已浏览数和生词数。
- 提供“继续学习”“顺序学习”“随机学习”和“生词本”入口。
- 顶部搜索框支持输入时即时显示结果。
- 搜索结果显示单词和精简中文释义；打开结果后进入词条详情，返回时保留原搜索词和滚动位置。

### 6.2 学习页

- 顶部显示返回、当前位置、阅读/回忆分段切换和收藏按钮。
- 单词、音标和朗读按钮始终可见。
- 阅读模式直接显示释义、等价词、例句和译文。
- 回忆模式先显示单词与音标；单击提示区域或按空格显示与阅读模式相同的详情。
- 切换模式不前进、不计为新浏览，也不重播发音。
- 进入新词时增加浏览次数并立即保存当前位置。
- 顺序模式按 PDF 源顺序循环；到达末尾后明确提示“已到最后一个词”，不会自动回到开头。
- 随机模式创建持久化随机队列，在队列用完前不重复；用户可主动“重新洗牌”。

键盘快捷键：

- `Left`：上一词。
- `Right`：下一词。
- `Space`：回忆模式显示或隐藏答案；阅读模式不拦截普通控件操作。
- `P`：朗读当前单词。
- `S`：收藏或取消收藏。
- `Ctrl+F`：聚焦搜索框。

### 6.3 生词本

- 按收藏时间倒序显示。
- 支持搜索和打开词条详情。
- 可在列表或详情中取消收藏。
- 空状态明确说明如何添加生词。

### 6.4 设置

- 选择默认阅读模式。
- 调整语速。
- 选择可用的英文语音。
- 提供“重置学习位置”和“清空全部本地数据”操作；清空操作必须二次确认。

## 7. 状态与数据流

1. 首次启动时验证内置内容数据库是否存在且结构版本正确。
2. 在用户数据目录创建或升级 `user.db`。
3. 首页从内容库读取统计信息，从用户库读取进度。
4. 打开学习页时，`StudySession` 恢复队列和当前位置。
5. 切换单词时查询只读词条，更新用户进度，再通知界面刷新。
6. 点击朗读时，仅把当前词头发送给 `SpeechService`。
7. 替换新版本 `.exe` 后，新程序继续读取原用户数据库；必要时执行向前兼容的模式迁移。

## 8. 异常处理

- **内容库缺失或版本不符：** 停止进入主界面，显示明确的重新获取应用提示，不静默创建空词库。
- **用户库损坏：** 把原文件重命名为带时间戳的备份，创建新库并提示进度已重置；不删除损坏文件。
- **语音不可用：** 禁用朗读按钮并显示非阻塞提示，其他功能继续工作。
- **没有英文语音：** 使用系统默认语音并提示可在 Windows 中安装英文语音包。
- **搜索无结果：** 显示空状态，不抛出错误。
- **保存进度失败：** 保留当前内存状态，提示用户检查磁盘空间或目录权限，并在下一次状态变化时重试。
- **意外异常：** 捕获到应用级日志文件；日志不含 PDF 全文、个人信息或遥测数据。

## 9. 测试与质量保证

### 9.1 解析器测试

- 覆盖首个词表页、中间页、末页、补充列表边界、多义词、换行等价词和中英文粘连案例。
- 验证所有词条都有非空词头、稳定主键、合法源页码和唯一源顺序。
- 验证解析词条总数、各列表计数和审计报告一致。
- 对异常标记进行人工逐条复核，并随机抽检至少 100 条普通记录。
- 将抽检修正记录保存为可重复应用的覆盖文件，避免修改解析器后丢失人工修正。

### 9.2 应用逻辑测试

- 顺序导航边界、随机队列无重复和重新洗牌。
- 阅读/回忆切换保持当前词，回忆答案状态在换词时重置。
- 收藏、取消收藏、搜索和进度持久化。
- 用户数据库首次创建、升级和损坏恢复。
- 语音可用与不可用两种路径。

### 9.3 界面与打包验证

- 验证窗口缩放、中文与音标显示、长释义换行和键盘焦点。
- 验证无鼠标时可完成学习、朗读和收藏。
- 使用临时用户数据目录启动打包后的 `.exe`，完成首页、搜索、双模式学习、生词本和重启恢复冒烟测试。
- 确认双击 `.exe` 不出现控制台窗口，不需要管理员权限，不读取原 PDF，不发起网络请求。

## 11. 验收标准

- 在 Windows 10/11 x64 上可直接双击启动，无控制台窗口和管理员权限要求。
- 应用不依赖网络、微信、原 PDF 或额外 Python 安装。
- PDF 正文与补充部分的全部已核验词条均可搜索和浏览。
- 阅读和回忆模式可随时切换，且保持当前词与收藏状态。
- 顺序与随机连续学习、生词本、搜索、进度恢复和 Windows 发音正常工作。
- 重启应用后恢复上次模式、浏览顺序和当前位置。
- 所有自动化测试通过，打包后的可执行文件通过冒烟测试。
